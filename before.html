Eventually I'd like to expand the Path to cover everything from the beginning, starting with basic computer science. For now, the scope of the path is limited to what I beliece is the area most in need to guidance: how to get from being a decent C programmer, to programming your own complete, commercial games. If you are not yet ready for step 1 of the Path, here are the things you should study. I'll add more resources here over time - feel free to send me recommendations.

Basics of electricity (voltage, resistance, capacitance)

  • Electricity: the Basics - ITP Physical Computing
  • Alpha Phoenix on YouTube - Learning About Electricity!

Basics of electronics (transistors, logic gates, latches, clock signals, binary numbers)

  • Ben Eater on YouTube - Making Logic Gates From Transistors
  • Build some simple breadboard circuits
  • Dive Into Systems - 4. Binary and Data Representation

How CPUs work (Von Neumann architecture, instruction execution)

  • CODE - The Hidden Language of Computer Hardware and Software
  • Ben Eater on YouTube - Building an 8-Bit Breadboard Computer
  • Dive Into Systems - 5. What von Neumann Knew: Computer Architecture

Basic assembly programming (doesn't matter which ISA)

  • Ben Eater on YouTube - Build a 65c02-Based Computer From Scratch
  • 6502.org Tutorials and Primers
  • HBC-56 - Troy Schrapel's custom 6502 computer, with an emulator and debugger
  • AppleWin - Apple II emulator and debugger - with ports to Linux and MacOS

Basic C programming

  • Beej's Guide to C Programming
  • Make a bunch of programs! Guess the number, hangman, snake, Hunt the Wumpus, anything you think of...

Memory management (static and dynamic allocation, bump allocators, arenas)

  • No resources to recommend just yet. Look up the topics and try different types of memory management in your programs.

Resources that cover several of the above topics

  • Nand to Tetris - Logic, electronics, CPUs, assembly
