Installing MacOS SDK, LLDB, GCC, Clang, CMake, and Git on MacOS

First, install Xcode Command Line Tools. Open a terminal and type:

xcode-select --install

Now install Homebrew:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Install GCC, Clang and CMake with Homebrew:

brew install gcc llvm cmake

Mac builds in an alias so that gcc redirects to clang. Several workarounds are available, but the easiest is just to use "gcc-XX" to invoke GCC. At time of writing, GCC is in version 15, so gcc-15 runs the installed (latest) gcc. You can also make sure GCC gets used by CMake for C compilation by prepending "CC=gcc-15" to your cmake command. You may also want "OBJC=clang" to keep using clang for Objective-C code.

Try typing "gcc --version" and you should get a Clang version back.

Check your installation with the following commands (press enter after each one and check the output):

gcc-15 --version clang --version cmake --version lldb --version git --version

At time of writing, Apple's XCode is not based on the latest llvm version 22, but some of my code requires this version or later. You've installed Clang via Homebrew, but make sure the output of "clang --version" is something like, "Homebrew clang version xxx," not "Apple clang version xxx." If yours reports an Apple Clang version, type this command into your terminal:

echo 'export PATH="/opt/homebrew/opt/llvm/bin:$PATH"' >> ~/.zshrc

Restart your terminal and check again.
